Funciones del sistema operativo y gestión de recursos

1. Gestión de recursos
El sistema operativo actúa como un administrador que asigna los recursos del hardware (CPU, memoria y dispositivos de entrada/salida) para garantizar que los programas funcionen de manera eficiente y sin conflictos.

Control de la CPU:

Asigna tiempo de procesamiento a cada tarea activa mediante un planificador de procesos .
Coordinar la ejecución de procesos para evitar sobrecargas o conflictos.
Ejemplo: Cuando abres varias aplicaciones, el sistema operativo decide cuál tiene prioridad para usar el procesador.


Gestión de la memoria:

Supervise el uso de la memoria RAM , asignando espacios específicos a cada programa en ejecución.
Evite conflictos entre programas que puedan intentar usar la misma porción de memoria.
Usa memoria virtual (en el disco duro) cuando la RAM no es suficiente.
Ejemplo: Al editar un documento y escuchar música, el sistema garantiza que ambos procesos utilicen la memoria correctamente.

Gestión de dispositivos de entrada/salida:

Coordina la comunicación entre dispositivos como teclados, ratones, impresoras y pantallas.
Proporciona controladores (drivers) que permiten que el hardware y el software trabajen juntos.
Ejemplo: Al conectar una impresora, el sistema operativo instala el controlador necesario para que funcione.

2. Interfaz de usuario
La interfaz de usuario permite al usuario interactuar con la computadora de manera sencilla y eficiente.

Interfaz gráfica (GUI - Interfaz gráfica de usuario):

Utilice elementos gráficos como ventanas, iconos, botones y menús para facilitar la interacción.
Ejemplo: En Windows, puedes usar el ratón para abrir carpetas o arrastrar archivos.

Interfaz de línea de comandos (CLI - Interfaz de línea de comando):

Requiere que el usuario introduzca comandos de texto para realizar acciones.
Más utilizado por administradores de sistemas y desarrolladores.
Ejemplo: En Linux, puedes usar el comando lspara listar los archivos de un directorio.

3. Gestión de archivos
Un sistema operativo organiza, almacena y permite el acceso a los datos en dispositivos de almacenamiento como discos duros, memorias USB o la nube.

Estructura jerárquica:

Los archivos se organizan en carpetas y subcarpetas para facilitar su localización.
Ejemplo: Una carpeta llamada "Trabajo" puede contener subcarpetas para "Proyectos" y "Presentaciones".

Operaciones básicas:

Crear, leer, escribir, copiar, mover y eliminar archivos.
Ejemplo: Al guardar un documento en Word, el sistema operativo almacena el archivo en el disco duro en la ubicación seleccionada.

Sistemas de archivos:

Determinan cómo se organizan y gestionan los datos en el almacenamiento. Ejemplos: NTFS (Windows), ext4 (Linux), APFS (MacOS).

4. Seguridad
El sistema operativo protege los datos y recursos del sistema frente a accesos no autorizados, errores y ataques externos.

Autenticación:

Requiere credenciales como nombres de usuario y contraseñas para permitir el acceso.
Ejemplo: Ingresar una contraseña al encender la computadora.

Permisos de acceso:

Restringe qué usuarios o aplicaciones pueden acceder, modificar o eliminar archivos.
Ejemplo: Un archivo en una red compartida puede ser accesible solo para lectura por ciertos usuarios.

Protección contra amenazas:

Detecta y bloquea programas maliciosos mediante herramientas de seguridad como firewalls o antivirus.
Ejemplo: Un firewall impide conexiones no autorizadas desde Internet a tu computadora.
